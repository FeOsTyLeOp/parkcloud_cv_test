# ParkCloud CV Concept: Анализатор занятости парковочных мест

Прототип системы компьютерного зрения (CV/ML) для децентрализованного анализа видеопотоков.

Проект решает "Пробную задачу" из ТЗ: принимает на вход изображение с камеры, калибровку камеры (матрицу перспективы) и разметку парковочных мест (GeoJSON), а на выходе формирует поток событий (JSON) с актуальной занятостью мест.

## Основные фичи и архитектура
* **CV-Детектор (YOLOv11):** В качестве базовой модели выбран современный YOLOv11n. Согласно бенчмаркам на парковочных датасетах (например, PKLot), 11-я версия дает наилучший баланс между FPS на Edge-устройствах и точностью распознавания плотно стоящих авто (по сравнению с v8/v9/RT-DETR).
* **Геометрия и гомография (OpenCV):** Внедрена математическая трансформация перспективы. Мы не анализируем Bounding Box целиком, мы берем только нижнюю центральную точку автомобиля (пятно контакта с асфальтом) и переводим её из координат матрицы камеры в координаты векторной карты.
* **Бизнес-логика (Shapely):** Проверка пересечений (Intersection) точек автомобилей с полигонами парковочных мест из GeoJSON файла.
* **Утилита разметки (Bonus):** Написан кастомный скрипт `annotator.py` для быстрой визуальной генерации правильного `park.geojson` по кликам мышки для любой камеры.

## Ограничения MVP и план развития (Production)

В рамках обобщения задачи до уровня Production, текущий концепт требует следующих доработок:

    Борьба с окклюзией (перекрытием): Настройка алгоритма Sensor Fusion - слияние данных о занятости одного машиноместа с нескольких перекрывающихся камер.

    Исключение мерцания и ложных срабатываний: Внедрение трекера (например, BoT-SORT или ByteTrack). Место должно помечаться занятым только если машина въехала в полигон и её скорость упала до нуля на N секунд.

    Адаптация к погоде (снег/дождь/ночь): Сбор датасета с конкретного объекта ParkCloud и Fine-tuning модели YOLOv11/аналога под железо.
